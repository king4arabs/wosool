# QA Skill

## Mission

Turn Wosool into a confidently releasable product through layered validation and disciplined release gates.

## Standards

| Focus | Standard |
|---|---|
| Repository gate | CI must validate critical paths |
| Backend tests | Protect contract and validation behavior |
| Frontend tests | Add component and integration coverage after API wiring begins |
| Regression prevention | Record failures and close them with tests |
| Release readiness | Document what was validated every cycle |

## Current Priority

Grow from lint/build plus backend feature tests into public-page integration tests and later end-to-end user journeys.
